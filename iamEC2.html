Identity and Access Management

IAM has a GLOBAL view and Contains:

  • Users, Groups, Roles
  • Permissions are governed by policies (JSON)
  • MFA can be set up
  • IAM has predefined 'managed policies'
  • It is best to give users minimal amount of permissions needed to perform their job.

IAM 101

ONE:

  • IAM User per Physical Person
  • IAM Role per Application

NEVER:

  • COMMIT IAM credentials
  • Write IAM credentials in code
  • share IAM credentials
  • User Root Account except for initial setup
  • Use ROOT IAM Credentials

EC2 Instance Types
R: Application that need a lot of RAM – in Memory caches
C: applications that needs good CPU – compute / database
M: applications that are balanced (think “medium”) – general /web application
I: applications that need good local I/O (instance storage) – databases
G: applications that need a GPU – video rendering / machine learning

EC2 Launch Types
OnDemand: Short work load, predictable pricing highest Cost no Long term commitment Billing - per second
Reserved Instance: Long workloads (>= 1 year) 75% discount to OnDemand reserve for 1 - 3 years Great for Database
Convertible Reserved Instances: Can change EC2 instance type up to 54% discount
Scheduled Reserved Instances: Launch within a time window you reserve When you require a fracrtion of Day / Week / Month
Spot Instances: Short workloads, for cheap, can lose instances 90% discount to OnDemand Most cost effiecient instance on AWS Good for Batch Jobs, Data Analysis, Image Processing No - databases of critical jobs
Dedicated Instances: No other Customers will share the instance Hardware is shared
Dedicated hosts: Book entire physical server, control instance placement visibility to Sockets and Physical Cores Good for regulatory or complaince needs and Bring Your Own Licence

Elastic Network Interface

EC2 Pricing

Prices vary depending on:
  • Region
  • Instance Type
  • On-Demand vs Spot vs reserved vs Dedicated Host
  • Linux vs Windows vs Private OS
  • Storage, Data Transfer, laod balancing, elastic IP
  • You are no charged when Instance is "stopped"
